Наверное, JDK 11 будет достаточно для компиляции и запуска

0.
Переименуйте comp.b в comp.bat

1.
Перетащите файл Kek.java на comp.bat, чтобы скомпилировать файл

2.
Если ничего не сломается, рядом появится файл Kek.class
Батник предложит запустить программу
Дальше - на свое усмотрение

3.
Если на момент компиляции Kek.class уже есть, то батник уточнит
"перекомпилировать, запустить или выйти?"

4.
Можно перетащить Kek.class на батник, чтобы сразу запускать

5.
Изначально программа предлагает на ввод числа в десятичном формате.
Чтобы запускать её для работы с двоичными числами:
Создайте рядом файл _args.txt
В нем напишите "binary" без кавычек
Сохраните и запускайте, как и раньше
Чтобы вновь работать с десятичными числами, удалите _args.txt

comp.bat написан мной и не подлежит распространению другими лицами.
